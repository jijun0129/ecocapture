### 📌 규칙

### 아래 커밋 컨벤션을 지키며 commit을 진행합니다.

```bash
<type>: <short summary>
```

1. `<type>` 에 들어갈 수 있는 항목들
   > 어떤 작업을 했는지 기록하는 용도입니다.

**사용 타입**

- Feat: 새로운 기능 추가
- Fix: 버그 수정
- Docs: 문서 관련
- Style: 스타일 변경 (포매팅 수정, 들여쓰기 추가, …)
- Refactor: 코드 리팩토링
- Chore: 그 외 자잘한 수정
- Comment: 주석 관련 작업
- Test: 테스트 관련
